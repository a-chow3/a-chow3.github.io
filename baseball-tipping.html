Pitch Tipping and 3D Pose Analysis

An end to end computer vision pipeline that turns broadcast angle video into 3D skeletal data, then uses ensemble modeling to detect the mechanical tells that give away a collegiate pitcher's next pitch.

Project overview

Pitch tipping happens when a pitcher unintentionally reveals their upcoming pitch through a subtle physical habit, a slightly different glove height, a longer pause, a change in posture during the set. This project uses deep learning and ensemble modeling to catch those signals. By converting 2D broadcast angle video into 3D skeletal data, it quantifies inconsistencies in the set position and hands the UVA staff a specific, actionable read before an opposing dugout finds the same tell.

Target audience

The UVA Baseball coaching staff. The whole point is to move from a vague worry to a named, fixable habit, backed by a signal strength and a p value so the staff can trust it and drill it.

Technical stack

Object detection uses YOLOv8 to localize the pitcher. Specialized glove detection and tracking runs through the Roboflow API. Pose estimation uses MediaPipe to pull a 2D skeleton. The modeling is a Random Forest ensemble with bagging and 200 estimators, and the deliverables are rendered in Matplotlib for the tipping dashboard and Plotly for interactive 3D skeletons.

Data structure and pipeline

The raw data is high resolution game footage. Because of platform and storage constraints, each aggregate video is capped at 50 pitches, each pitch is isolated into a 7 second clip that focuses on the pre delivery and set positions, and everything is managed through OneDrive for cross team collaboration. The directory is organized by pitcher, with per pitch clips and a summary table:

TruMedia Video/ [Pitcher_Name]/ uva_[name]_pitch_1.mp4 uva_[name]_pitch_2.mp4 uva_[name]_summary_df.csv Processed/ [Metadata and JSON detection logs]

The pipeline runs in five stages:

  1. Pitcher localization. A custom region of interest is defined, then YOLOv8 finds every human figure and locks onto the pitcher based on proximity to the search window centroid.
  2. Glove detection and state logic. The Roboflow API tracks the glove. The set position is determined algorithmically when the glove holds spatially stable for 45 consecutive frames, roughly three quarters of a second.
  3. 3D projection. The 2D coordinates are projected into 3D. I use the scale of the glove's bounding box as a depth proxy for the Z axis, and normalize using the pitcher's height and standard human body proportions.
  4. Ensemble learning. A Random Forest analyzes the XYZ joint coordinates during the set position to classify the pitch type.
  5. Signal detection. I extract feature importance to identify which specific body parts, for example shoulder tilt or glove depth, are tipping the pitch.

Deliverables

The first deliverable is the tipping dashboard below: a single visual report with model accuracy against baseline, a body part radar showing which muscle groups are most inconsistent, and categorized signal flags. The second is an interactive 3D skeleton map in Plotly that lets a coach rotate and inspect a pitcher's posture from angles the single broadcast camera never captured.

Future work

Next steps are temporal sequence modeling with LSTMs or transformers to read the motion of the whole windup rather than just the static set, a lightweight edge version for real time dugout use, and batter correlation that ties identified tips to higher exit velocities for opponents using TruMedia hit data.
