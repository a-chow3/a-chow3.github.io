Swing and Stance Clustering

Group hitters into swing and stance archetypes, learn how each archetype performs against every pitch, then layer on predictive models that classify a hitter, forecast run value, and map the path to the next tier.

Role
Analyst, end to end build
Stack
Python · scikit learn · pandas · matplotlib
Audience
Coaches, players, player development
Data
Baseball Savant (public)
The archetypes below are my real findings from the June 9, 2025 snapshot. The scatter positions and model outputs use dummy values for public display, but the archetype names, summaries, and weaknesses are the genuine article. Full notes live in the public repository.

Purpose

Two hitters can post the same batting line and be nothing alike. This project groups hitters by how they actually move, their stance and swing mechanics, to surface distinct hitting archetypes. Once you can name a hitter's type, you can tailor pitch sequences and matchups against it, and you can coach toward it.

Target audience

Coaches and player development staff who want a shared vocabulary for swings, and analysts who want a defensible way to segment hitters that goes beyond raw results.

Methods

I scraped stance, swing, batted ball, and value data from Baseball Savant and merged it into one player table. After cleaning and standardizing the mechanical features, I fit Gaussian Mixture Models and compared them against K Means, using PCA to project the high dimensional swing space into two readable dimensions. Each cluster gets a reference profile of mean and spread for every metric. From there I built pitch level analysis, a vulnerability matrix, and three predictive models described below.

Results

The June 9, 2025 run produced ten swing archetypes, each with a clear identity, a scouting weakness, and a plan of attack. Together with the predictive models, they turn a wall of numbers into a language a coach can use in a hitters meeting.

PCA view of swing clusters

How to read it: each dot is a hitter, placed by a two dimensional PCA projection of their swing mechanics. Color is the archetype the model assigned. Tight, separated colors mean the archetypes are real rather than arbitrary cuts. Swings sit in this space, not stances, because swings vary far more. Archetype profiles

Each card is a real archetype from the June 9 run, with its summary and the scouting weakness that drives a plan of attack.

Vulnerability matrix

Run value per 100 pitches for each archetype against each pitch family. Positive favors the hitter, negative favors the pitcher.

How to read it: find the archetype, then scan its row for the coldest cell. That is the pitch family to lean on. The matrix lines up with each archetype's written weakness, so it is the bridge from a named cluster to a concrete attack plan a coach can call.
Part 2 · Predictive modeling

Three models, three jobs

This is the part I most like to show, because it moves from describing hitters to predicting and directing. Every output below is a dummy run in the real format.

Model 1 · Swing archetype classifier demo

A Random Forest that reads a hitter's mechanics and results and returns their most likely archetype. A KNN imputer (k = 5) fills gaps using the five most similar players.

Feature importance
How to read it: the bars rank which inputs drive the classification. The console shows a single prediction: the model's top archetype for a test hitter and the probability spread across the others. A high top probability means a clean, confident type. A flatter spread means a hybrid swing.

Model 2 · Run value by pitch type demo

A Gradient Boosting Regressor that estimates the expected run value of a pitch type against a hitter. Trained on an 80 / 20 split with 500 estimators, a 0.02 learning rate, and max depth 6.

Predicted vs actual run value
How to read it: I hand weighted the inputs so wOBA and slugging carry the most signal and noisier rates like whiff carry less. Points near the diagonal line mean the model's predicted run value matched what really happened. The tighter that cloud, the more you can trust the pitch level forecast.

Model 3 · Tier progression demo

Players sit in six tiers from Poor to Superstar. This model answers a forward looking question: what should player X change to reach the next tier? It compares a hitter to those who already succeed above them using t tests and Cohen's d.

How to read it: each row is an adjustment that most separates the target tier from the current one, ranked by effect size. A larger Cohen's d means a bigger, more reliable gap. These are starting points for training focus, not promises, since every hitter is different. Random Forest and Gradient Boosting rank the signal, and a multinomial Logistic Regression turns it into readable benchmarks for staff.
