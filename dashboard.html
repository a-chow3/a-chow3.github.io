UVA MBB Performance Dashboard

A daily decision support tool that turns raw practice and game tracking data into a weighted player efficiency index, shooting analytics, and head to head comparisons, built to help coaches make faster, evidence based decisions on the court and in the lineup.

Purpose

UVA's staff charts every practice and game with a detailed action tagging system, dozens of tracked events per possession, from a good wall on defense to live ball turnovers. Raw, that data is unreadable. The dashboard's job is to compress it into a single, trustworthy answer to the question coaches actually ask: who is helping us win, and in exactly which ways?

Target audience

Coaches and performance staff who need answers in seconds between drills, not analysts. Every view is built to be glanceable, with color coded indices, team relative percentiles, and plain language tiers instead of raw z scores.

Methods

1 · A weighted action value model

Each of the roughly 60 tracked actions carries a point weight that reflects its impact on winning. I keep two weighting systems that the user can toggle live: an original expert tuned set, and an EPA set (expected points added) calibrated to possession outcomes. A player's offensive and defensive scores are the weighted sums of their actions.

2 · Team-relative standardization

Raw scores don't travel across sample sizes, so I standardize each player's offensive and defensive score into z scores against the team, then map those to an intuitive Performance Index (100 is team average, and each standard deviation is worth about 34 points) using the normal CDF. Players then fall into tiers that run from Very Poor all the way up to Elite.

3 · Shooting efficiency layer

A separate module computes True Shooting %, Effective FG%, FG%, and points per shot, each benchmarked against the team average, plus a make and miss breakdown by shot type and a sortable team leaderboard.

Design principle: everything is filterable by date. Coaches can isolate a single practice, a road trip stretch, or the full season, and every index, percentile, and chart recomputes against only the selected events.

Process

  • Built an automated pipeline landing daily tagged practice & game data into cleaned Snowflake tables.
  • Engineered the token to weight to category mapping, and the z score to index conversion, in pandas.
  • Designed a dark, glanceable Streamlit interface with live weight toggling, event filtering, and drill downs.
  • Iterated directly with coaching staff to tune weights, color thresholds, and which views mattered most.

How the Index is calculated
1. Token counting. Each action is counted per player from the selected events.
2. Weighted scoring. Tokens times weights, positive for good actions and negative for errors.
3. Z scores. Offensive and defensive scores standardized against the team.
4. Index. Z scores become a percentile index using the normal CDF, where 100 is average and one standard deviation is about 34 points.
