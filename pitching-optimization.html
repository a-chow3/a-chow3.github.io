Pitching Optimization & PES

A run value engine that scores every pitch, clusters pitches by matchup, and trains decision trees that show the exact point where a pitch flips from helping the pitcher to helping the hitter.

Role
Analyst
Stack
Python · scikit learn · R Shiny
Audience
Pitchers and pitching coaches
Data
UVA game tracking (proprietary)
All figures use dummy data. The Pitch Efficiency Score, the clustering, and the decision tree logic are the real design. UVA's underlying tracking data is proprietary and is not present here. Code lives in the public repository.

Purpose

The core problem is putting a trustworthy number on the value of a single pitch. Results alone lie to you, a well executed pitch can still get hit. I built a run value attribution system that scores every possible pitch outcome, rolls it into a Pitch Efficiency Score, and then uses that score to find what a genuinely good pitch looks like in a specific matchup.

Target audience

Pitchers and the pitching staff. The end product is a benchmark a pitcher can train toward, expressed as concrete thresholds in velocity, movement, and location rather than abstract math.

Methods

I start from a hard coded run value for every pitch result, then compute a Pitch Efficiency Score for each pitch. I group pitches by pitcher handedness against batter handedness and by pitch type, and cluster within those groups to find profiles of successful pitches. For each matchup and pitch type I fit a decision tree on the pitcher's average metrics, which reveals the thresholds where a pitch stops working. An earlier exploratory pass used K Means to cluster MLB pitchers by value and study how a handful of average bodied arms produced elite results.

Results

The output is a set of readable, matchup specific benchmarks: for example, in a lefty on righty look, sliders buried down and in beat sliders left over the plate, which drift into the hitter's barrel path. Coaches get a clear target for execution instead of a vague instruction to throw a better slider.

Pitcher value clusters (MLB EDA)

How to read it: K Means split MLB pitchers into four value tiers using expected and actual performance. The top cluster is highlighted. Within it, I zoomed in on arms that produce elite results without the height and weight advantages, to learn what the rest of a staff can actually copy.
Deliverable 2 · dummy data

Pitch Efficiency Score leaderboard

How to read it: PES rolls each pitch's run value impact into one number. Higher is a more efficient pitch. Because it scores process and not just the box score, a pitch that was hit hard but perfectly located still grades well, which is exactly what you want when coaching execution.
Deliverable 3 · dummy data

Matchup decision tree

How to read it: follow the branches from the top. Each split is a metric threshold. A blue leaf is a spot where the pitch favors the pitcher, an orange leaf favors the hitter. The path to a blue leaf is the execution target: hit these velocity, movement, and location marks and the pitch plays.
